# 技術者面接 / 現場面接 (peer) — 一般解 20 問

担当: 入社後に同僚になる現場の人 (先輩 / 同チームメンバ)。 評価対象は **実装力
/ 協働力 / チームに馴染むか / 自分の領域で「使える」 か**。 全職種共通の形で
書いてあるが、 質問は「自分の領域での」 と読み替えること。

## 001 — 直近のプロジェクトでの担当範囲を教えてください

**axes**: [clarity, self_understanding, target_fit]

**浅い回答 (NG)**:
- 「全部やりました」 (具体性なし)
- 役割の上下を語らない
- チームでの自分の比重を曖昧にする

**評価される回答の枠組み**:
**①プロジェクト概要 (規模 / 期間 / 人数) ②自分の担当領域と境界 ③チーム内での
役割 (リード / メンバ / 補助) ④他職種との接点**。 「全部」 ではなく「自分が一次
責任を持った領域」 と「補佐した領域」 を分ける。

**深掘り例**:
- 担当外で関わった部分は?
- 他の人がやった部分との境界はどう決めた?

**評価される回答の枠組み**:
**①課題の具体内容 ②自分の最初の仮説 ③仮説の検証 ④採った打ち手 ⑤結果と学び**

見える。 苦労 = 「時間がかかった」 ではなく **意思決定が難しかった** 例を選ぶ。

**深掘り例**:
- もう一度同じ課題に当たったら、 何を最初に変える?
