# stage/ — 面接タイプ別の一般解 QA

職種 (planner/programmer/designer/sound) と直交する軸として、 **面接タイプ
ごとに頻出する質問** をまとめる。 by-role は「何を聞かれるか (技術領域)」、
by-stage は「誰が何のために聞くか (動機)」 を扱う。 セッション開始時、
`(role + stage)` 両軸で RAG 検索する想定。

---

## stage タグの意味

| stage | 面接担当 | 主な評価軸 |
|---|---|---|
| `hr` | 人事・採用担当 | 人柄、 モチベ、 カルチャーフィット、 退職リスク、 待遇整合 |
| `peer-tech` | 同職種の現場担当 (先輩 / メンバ) | 実装力、 協働力、 自分の現場での読み |
| `lead-tech` | 部長・テックリード・マネージャ | 中長期判断、 リーダーシップ、 育成、 採用関与 |
| `final` | 社長・役員・CTO | ビジョン、 業界観、 経営目線、 入社の覚悟 |

LUDIARS 規約上、 各社の慣習による「順番」 「人数」 「省略」 は別データに切り出す
(本ファイルは「質問の傾向」 だけを扱う)。

---

## 質問の構造

by-role と同じ:

1. **axes** — 6 軸ラベル (`consistency` / `clarity` / `demeanor` /
   `self_understanding` / `target_fit` / `depth_resilience`)
2. **浅い回答 (NG パターン)**
3. **評価される回答の枠組み** (構造)
4. **深掘り質問例**

---

## stage と role の使い分け

- by-role: 質問の **題材** が職種特有 (例: 「シェーダー書いたことある?」 はプログラマ)
- by-stage: 質問の **目的・深さ** が面接段階に紐付く (例: 「失敗から学んだことは?」
  は HR でも final でも来るが、 期待される深さが違う)

RAG 検索クエリ例:
```
role:programmer stage:lead-tech → 「テックリードがプログラマに聞く質問群」
role:planner    stage:final     → 「役員がプランナに聞く質問群」
```
