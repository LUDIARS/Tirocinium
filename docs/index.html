Tirocinium — tīrōcinium

新兵訓練、初任修練。本番前の本気の試行錯誤。
本人の ES / ポートフォリオ / 過去面接問答 を教師データに持つ AI を相手に、 音声対話で面接を反復練習し、Sonnet ストリームで応答しつつ 複合 LLM が適度な分量で ペルソナ評価を返す、LUDIARS の面接練習アプリ。

サービスの役割

Tirocinium 自体は「予約台帳 + 集計 + LLM オーケストレータ」に徹する。 個人データ (履歴書 / ポートフォリオ / 面接ログ) は本サービスの責務範囲外で、 認証は Cernere、音声は Imperativus、生データの永続化は Memoria に委譲する。 自リポが DB に持つのは「セッションメタ + 評価結果 + 公開企業情報」だけ。

🎯本番に近い圧

就活初学者が本番に近い圧で面接練習を反復できる。面接官ペルソナの pressure が深掘りの粘りと沈黙の使い方を制御。

🧬本人特化

教師データを本人固有に持つ (汎用面接 bot ではない)。fine-tune せず RAG + 弱点プロファイルで本人モデルを育てる。

🎙️低遅延の音声対話

音声で投げて音声で返す対話を遅延小で回す。Sonnet 応答は TTS と並行にトークン単位で流す。

📊邪魔しない評価

評価は応答ストリームを邪魔せず、別レイヤ (Opus) から適度な分量で返す。6 軸スコア + 改善 hint。

🏢企業探索

ES からおすすめ企業を返し、公開情報をクロールして企業プール・ゲームグラフ・OB 実績を育てる。

🚪裏口 (卒業生)

卒業生の自己投稿面。Discord マジックリンク認証で在籍企業・学生/業界向けメッセージを残す。

2 つのモード

💻ローカルモード

ユーザ PC 上のみで完結。ネット切れても動く。LLM は軽量ローカルモデル (ollama 等)、STT/TTS も local Whisper / piper を想定。教師データはローカル SQLite。評価層は OFF または簡易。

☁️サーバーモード

LUDIARS サーバ群を使い複合 LLM (GPT-5.5 + Opus + Sonnet) で本番面接体験。認証は Cernere、音声は Imperativus、生データ永続化は Memoria を検討。枠が埋まると予約に誘導。

起動時にサーバ到達性 (/health) と認証トークン有効性で自動判定。失敗すればローカルへフォールバック。明示切替 UI もある。

本人特化 = RAG + 弱点プロファイル (fine-tune せず)

RAG (本人素材の検索注入)

ES / portfolio / 過去 Q&A は Memoria 側で永続化 + embedding。Tr は training_data_refs で参照だけ握る。session 開始時に (志望 tag + 弱点軸) を query に Memoria vector search → system prompt に注入。

弱点プロファイル

Opus 評価を user 単位で EMA 集約。6 軸の平均 + 分散 + 最弱 top-3 + hint 履歴を保持。session 開始時に「鍛えるべき軸」を抽出して system prompt に注入。

ペルソナ評価の 6 軸

主張一貫性論旨明快さ態度・落ち着き 自己理解志望適合深掘り耐性

評価は 0-5 スコア + 一言コメント + 改善 hint で返す。

5 段重ね system prompt

① 静的 root → ② 面接官ペルソナ → ②.5 弱点プロファイル → ③ session 開始時 RAG (role + stage tag) → ④ GPT-5.5 補正。 pressure は「聞き返しの粘り」「沈黙挿入」に変換、tics はそのまま転記。

予約フロー

サーバーモードは同時面接数が計算資源 (LLM コンテキスト数 + GPU 枠) で制限される。 枠が埋まったら待たせず、予約として時間を確定して帰す。

  1. ゲート判定 — 認証 OK → 同時面接 quota 内 → GPU/LLM プール空き、を順にチェック。全 OK なら即時開始。
  2. 予約 offer — 30 分単位 slot。最短で空く slot を slot_start ASC で走査して offer。1 ユーザ同時 1 件まで。
  3. 通知 — slot start 時に WS push + Nuntius 通知。開始 15 分前にも push。5 分以内に開始しなければ no-show で解放。

データ責務境界

LUDIARS ルール: 個人データは Cernere 単一情報源、自前 DB に持たない。

Tr が持つ

session メタ (id, user_id, start/end, mode, slot) / evaluation score (集計値) / reservation slot 状態 / training_data の参照 (id + Memoria URI) / 企業の公開情報 + おすすめ結果

Tr が持たない

ES / ポートフォリオ生データ / 面接トランスクリプト生データ (Memoria 永続化) / ユーザ氏名・連絡先 / training_data 本体 / ES から導出した本文の逐語コピー
