仕様 ↔ コード レビュー

spec/ 配下の AIFormat 構造化仕様と DESIGN.md を、実際の実装コード (apps/ / packages/ / scripts/) と突き合わせた対応レビュー。 凡例: ✅ 実装一致 · 🟡 部分実装 / 差異 · ❌ 未実装 · 📄 仕様のみ

総評

仕様と実装は 概ね一致している。面接コア (LLM オーケストレータ・推論エンジン・ペルソナ・予約・スキーマ)、 3 面構成 (本体/面接/裏口) と Bot A/B 分離、企業ドメインの大半は仕様通り実装済み。 一方で 外部依存待ち (GPT-5.5 / Iv TTS) と 後続 PR 待ち (FT-loop 起動) の未完部分、 そして 仕様にあるが未着手の機能 (面接質問プール) が残る。 逆に、仕様に明示されないまま先行実装された機能 (アナリティクス・技術グラフ・地図など) もある。

主要な一致

複合 LLM (役割別モデル割当・collapse)・5 段重ね system prompt・6 軸評価・phase 状態機・弁証法プローブ・予約 slot 確保 + 行ロック・裏口認証・Notion 取込・secret hydration。

主な gap

GPT-5.5 は gpt-4o 代替で待機 · prompt cache 未実装 · Iv TTS 結線 TODO · VAD は暫定 (energy 版) · FT-loop 起動は 501 · gBizINFO の URL 検索ステップ未確定。

未実装の仕様

spec/feature/companies/interview-questions.md の面接質問プール (テーブル + API) が一式欠落。面接質問生成の最優先素材として仕様化されているが供給経路がない。

5. 仕様外の先行実装 (コードのみ)

仕様に明示されないまま実装された機能。多くは本体面の運用・企業探索を支える付加機能。

  • アナリティクス — routes/analytics.ts + packages/analytics + analytics-viewer/ (page_view / company_view、日次・トレンド)。
  • 技術グラフ — GET /companies/by-tech + migrations/010_tech_platform.sql (企業×技術)。
  • 地図 / ジオコード — companies/geocode.ts + migrations/015_geocode_cache.sql + Google Maps マーカー。
  • ユーザ提供リンク分類 — POST /companies/:id/contribute (companies/contribute.ts)。
  • 自動 enrich キュー — companies/enrich-queue.ts (GET /companies/enrich-queue/status)。
  • Wikidata 連携 — companies/wikidata-*.ts (公式サイト URL 補完)。
  • Resources — routes/resources.ts (参考リンク + 卒業生メッセージ合成)。
  • 通知 stub — Nuntius URL 未設定なら no-op。
